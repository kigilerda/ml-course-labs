# Лабораторная работа 3

## Описание задания

Вам предоставлен набор данных с информацией о клиентах банка.

### Описание признаков

- `client_id` — идентификатор клиента (неотрицательное целое число)
- `Age` — возраст (вещественное число)
- `Income` — месячный доход (вещественное число)
- `BalanceToCreditLimit` — отношение баланса на кредитной карте к кредитному лимиту (вещественное число)
- `DIR` — Debt-to-Income Ratio, отношение долга к доходу (вещественное число)
- `NumLoans` — количество займов и кредитных линий
- `NumRealEstateLoans` — количество ипотек и займов, связанных с недвижимостью (натуральное число)
- `NumDependents` — количество членов семьи на содержании клиента, не включая самого клиента (натуральное число)
- `Num30-59Delinquencies` — количество просрочек по кредиту от 30 до 59 дней (натуральное число)
- `Num60-89Delinquencies` — количество просрочек по кредиту от 60 до 89 дней (натуральное число)
- `Delinquent90` — целевая переменная: наличие просрочек по кредиту более 90 дней (бинарный признак)

## Цель работы

Решить задачу бинарной классификации и сравнить качество различных моделей, включая дерево решений и ансамблевые методы.

## Необходимо выполнить

### 1. Анализ и предобработка данных

1. Провести разведочный анализ данных (EDA):
   - изучить структуру датасета;
   - проверить типы данных;
   - исследовать распределения признаков;
   - выявить пропуски, выбросы и аномальные значения;
   - проанализировать зависимости между признаками.

2. Выполнить предобработку данных:
   - обработать пропущенные значения;
   - удалить или скорректировать некорректные данные;
   - при необходимости преобразовать признаки.

3. Выполнить масштабирование или нормализацию данных.

4. Подготовить данные для обучения моделей:
   - выделить матрицу признаков `X` и целевую переменную `y`;
   - разделить выборку на обучающую и тестовую.

### 2. Дерево решений

1. Обучить модель `DecisionTreeClassifier`.

2. Визуализировать построенное решающее дерево.

3. Исследовать влияние глубины дерева на качество модели:
   - обучить дерево решений с глубиной от `1` до `11`;
   - оценить качество моделей;
   - определить момент начала переобучения;
   - выбрать оптимальную глубину дерева.

4. Подобрать оптимальные гиперпараметры дерева решений с помощью `GridSearchCV`.

### 3. Ансамблевые методы

#### Bagging

1. Выбрать любую классическую модель, не являющуюся ансамблевой.

2. Построить ансамбль с использованием `BaggingClassifier`.

3. Подобрать оптимальные параметры:
   - базовой модели;
   - `BaggingClassifier`;
   - с использованием `GridSearchCV`.

#### Stacking

1. Выбрать несколько классических моделей, не являющихся ансамблевыми.

2. Построить ансамбль с использованием `StackingClassifier`.

3. Подобрать оптимальные параметры:
   - базовых моделей;
   - стекинга;
   - с использованием `GridSearchCV`.

#### Random Forest

1. Обучить модель `RandomForestClassifier`.

2. Подобрать гиперпараметры случайного леса с помощью `GridSearchCV`.

#### XGBoost

1. Инициализировать параметры и обучить классификатор `XGBoost`.

2. Учесть, что `XGBoost` работает только с числовыми признаками.

3. Оценить важность признаков с помощью `F score` в `XGBoost`  
   (`F score`, а не `F-мера`!).
